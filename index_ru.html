Функциональный тестировщик

  • Junit. Реализовано динамическое создание пользователя с помощью Junit Extension. Для каждой роли пользователя назначен свой экземпляр браузера для последующей работы с контекстозависимой POM и отчетностью. Реализованы soft assertions для сбора ряда проверок и отчета о них в конце теста.
  • RestAssured. Повторение user stories на уровне API, дла формирования предусловий. Реализован state transition подход на уровне API - следуя бизнес-логике описаны ключевые состояний системы с помощью цепочки запросов API.
  • Selenide. Реализована гибкая структура POM для описания различных пользовательских ролей и различных состояний системы. Создание контекста страниц/компонентов для каждой роли. Следуя бизнес-логике, описан переход состояния UI-компонентов (набор контролов) для каждого заданного состояния в контексте различных ролей.
  • Allure Reports. Интеграция отчетов для отображения результатов тестов в Jenkins.
  • Jenkins. Настройка Jenkins для проекта. Удаленное выполнение тестов в Jenkins.
